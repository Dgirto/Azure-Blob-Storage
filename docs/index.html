¿Qué es y para qué sirve?

El conector Azure Blob Storage permite que el agente de Ruvic gestione blobs en un contenedor específico de Azure Storage. Una vez configurado, puedes pedirle al agente cosas como:

  • “Sube este reporte al contenedor con el nombre reportes/julio-2026.csv.”
  • “¿Qué archivos hay en la carpeta facturas/ del contenedor?”
  • “Descarga el archivo contrato.pdf y muéstrame su contenido.”

Operaciones soportadas:

Operación Qué hace
blob.list Lista los blobs del contenedor, con filtro por prefijo
blob.upload Sube (o sobrescribe) contenido en un blob del contenedor
blob.download Descarga el contenido de un blob
Este conector escribe blob.upload sobrescribe el blob del nombre indicado. El conector opera sobre un único contenedor; se recomienda que la clave/SAS del usuario esté limitada a ese contenedor específico.

Prerrequisitos

  • Una cuenta de Azure con una cuenta de almacenamiento y un contenedor ya creados.
  • Acceso al Azure Portal para obtener la clave de acceso o generar un SAS token, solo durante la configuración inicial.
  • El nombre de la cuenta de almacenamiento y del contenedor.

Configuración en el fabricante

Vas a obtener las credenciales de acceso al contenedor que quieres exponer a Ruvic. Se recomienda restringir el acceso a ese contenedor específico cuando sea posible.

  1. Localiza tu cuenta de almacenamiento y contenedor.

    En Azure Portal, busca "Cuentas de almacenamiento" → selecciona la tuya → "Contenedores" → confirma el nombre del contenedor que quieres exponer (o crea uno nuevo).

  2. Obtén la clave de acceso o genera un SAS token.

    Opción simple: en la cuenta de almacenamiento → "Claves de acceso" → copia key1 (da acceso a toda la cuenta). Opción recomendada para producción: en el contenedor → "Generar SAS" → permisos Read, Write, List → copia el token generado (restringido a ese contenedor).

    [ Captura: generación de SAS token con permisos Read/Write/List — reemplazar por assets/img/azure-blob-sas.png ]
  3. Verifica el acceso desde la línea de comandos.

    Con Azure CLI configurado:

    az storage blob list --account-name TU_CUENTA --container-name TU_CONTENEDOR --account-key TU_CLAVE

    Si responde sin error (aunque el contenedor esté vacío), el fabricante quedó listo. Anota estos valores para la siguiente sección: nombre de cuenta, clave/SAS y nombre de contenedor.

Permisos requeridos

Principio de mínimo privilegio: el usuario de Ruvic solo necesita estos permisos sobre el contenedor configurado. No otorgues permisos de administración de la cuenta (crear/eliminar contenedores, cambiar políticas de acceso).

Permiso Para qué sirve
Read Descargar blobs (blob.download)
Write Subir blobs (blob.upload)
List Listar los blobs del contenedor (blob.list)
Nota sobre la clave de cuenta Si usas la clave de cuenta (key1/key2) en vez de un SAS token, esa clave da acceso a TODOS los contenedores de la cuenta, no solo al configurado en el conector. Para limitar el alcance real, usa un SAS token restringido al contenedor.

Configuración en Ruvic

Entra a Settings → Conectores, busca la tarjeta Azure Blob Storage y pulsa Configurar. Diligencia el formulario con los valores que obtuviste en la sección anterior:

Campo del formulario Valor
Nombre de la cuenta de almacenamiento El nombre de la cuenta del paso 1
Clave de acceso La clave o SAS token del paso 2. Se guarda cifrada como secreto: la interfaz no volverá a mostrarla
Contenedor Nombre del contenedor a exponer
Timeout de conexión Opcional; 10 segundos por defecto

Guarda y pulsa Probar conexión — la plataforma verificará el acceso al contenedor configurado.

[ Captura: formulario del conector Azure Blob Storage en Settings → Conectores — reemplazar por assets/img/azure-blob-form.png ]

Verificación

  1. Test de conexión exitoso.

    El botón Probar conexión debe responder: “Conexión exitosa al contenedor 'mi-contenedor-produccion' de 'miempresastorage'”. La tarjeta del conector cambia a estado Configurado.

  2. Automatización mínima de prueba.

    Abre una conversación nueva con el agente y pídele: “Sube un archivo de prueba al contenedor con el texto 'hola mundo' y luego dime qué archivos hay.” El agente debe subir el blob y listarlo de vuelta. Si lo hace, el conector quedó operativo de punta a punta.

Solución de problemas

Mensaje / síntoma Causa probable y solución
“Credenciales inválidas o sin permiso suficiente” Clave o SAS token mal copiado, o el SAS expiró (los tokens SAS tienen fecha de vencimiento). Reintenta el paso 2 de la sección 3 y genera un token nuevo si expiró.
“No se pudo conectar a la cuenta … (timeout)” Nombre de cuenta mal escrito, o hay un problema de red saliente hacia Azure. Verifica el nombre exacto en el Azure Portal.
“El contenedor no existe” Nombre de contenedor mal escrito en el formulario. Verifica con az storage container list --account-name TU_CUENTA.
El agente lista blobs pero falla al descargar uno Falta el permiso Read en el SAS token, o el nombre del blob tiene mayúsculas/espacios distintos a los mostrados.
El conector aparece “Sin configurar” tras guardar Algún campo obligatorio quedó vacío. Ábrelo de nuevo y completa nombre de cuenta, clave de acceso y contenedor.

Preguntas frecuentes y notas de seguridad

¿El agente puede acceder a otros contenedores de mi cuenta?

Depende de qué credencial uses. Con un SAS token restringido al contenedor (recomendado), no. Con la clave de cuenta (key1/key2), técnicamente sí tiene acceso a todos los contenedores, aunque en la práctica el conector solo opera sobre el contenedor configurado en el formulario.

¿Dónde queda guardada la clave de acceso?

Como secreto en el almacén de credenciales de la plataforma, asociado a tu usuario. La interfaz nunca vuelve a mostrarla; solo indica “definida”. Para cambiarla, escribe una nueva en el formulario; para conservarla al editar otros campos, deja el campo vacío.

Rotación de credenciales

Si usas la clave de cuenta: regenera key1/key2 desde el Azure Portal periódicamente (esto invalida la clave anterior de inmediato). Si usas un SAS token: genera uno nuevo antes de que expire el actual y actualízalo en el formulario del conector.

¿Qué hago si las credenciales se comprometen?

  1. Regenera la clave de cuenta comprometida en Azure Portal, o revoca el SAS token (requiere una política de acceso almacenada, o esperar a que expire).
  2. Revisa los logs de acceso del Storage Account (Diagnóstico → Logs).
  3. Actualiza la nueva credencial en el formulario del conector.

¿Expiran las credenciales?

Las claves de cuenta no expiran solas (hay que rotarlas manualmente). Los SAS tokens sí tienen fecha de expiración definida al generarlos — planifica su renovación antes de esa fecha para que el conector no deje de funcionar.

Buenas prácticas Prefiere un SAS token restringido al contenedor específico en vez de la clave de cuenta completa; define una fecha de expiración razonable en el SAS; no expongas contenedores con datos sensibles que el agente no necesite; y evita compartir la clave o el token por canales no seguros.
